HTML Tags

A basic HTML tag has the following syntax: <tag>content</tag>. Most tags <tag> will have a corresponding closing tag </tag>.

Certain tags don't need a closing tag, like <br> and <meta>. In older code, they may be writte as <br />. This is because older versions of HTML required closing tags for everything, so this syntax indicates a "self-closing tag", where the tag is both an open and close tag. Just something to keep in mind, it's not necessary for HTML5

Tags can be nested. A nested tag is typically called the child, with the higher tag being the parent. Tags at the same nesting level are called siblings. Keep in mind that child tags must close before parents do.

Tags can also have attributes to them, which can add additional info to the tag's content. The syntax is <tag attribute="value">content</tag>. Some common global attributes are:

  • class useful for CSS
  • id useful for JS
  • contenteditable allows users to modify content
  • lang and dir help specify the language and direction of text flow that the element's content contains

Content Models

In the beginning, there were generally 2 types of tag models: block and inline. Block models appeared one after the each other, while inline models would appear in the middle of the content

<p>The paragraph tag is a block model.</p>

<p>Notice that each paragraph is a separate block.</p>

<p>However, for inline models like bolding, italics, and code, they don't disrupt the flow and formatting of the rest of the content</p>

Today, HTML5 has expanded to 7 models:

  • Flow: Elements included in normal flow of the document. Huge category that pretty much encompasses the rest of the categories
  • Metadata: Sets up the document presentation and behavior
  • Embedded: Imports external resources to the document, like vidoes, links, and images
  • Interactive: Content that's intended for any kind of user interaction, like buttons or drop down menus
  • Heading: Defines the header of a section, only includes tags <h1> - <h6>
  • Phrasing: Marks up text within content. Basically the former inline model with bolding, italics, and code
  • Sectioning: Defines the scope of headers and footers of a document

Behind-the-Scenes

<!-- btw this is what a comment looks like -->

Every HTML document needs a doctype tag at the very top of the document. This indicates to the brower that the incoming document is HTML. We can also specify different HTML versions, with the default being HTML5

<!DOCTYPE HTML>

After doctype is the <html> tag. This tag encompasses the entire document (besides doctype). The html tag has a lang attribute. Indicating the language of the document is very helpful for services like spellchecker, translators, and screen readers. To set document to English, use:

<html lang="en">

Next up is the <head> tag. The head tag contains all the document's metadata. Some example meta tags are:

<meta charset="utf-8">
Indicates the character encoding for the document. Ensures that the browser renders special characters and symbols properly

<meta name="description" content="insert description">
Provides a small description for the HTML document. Useful for indexing by search engines or yourself

<title>Document Title</title>
Sets the title of the document (shows up on the tab name in Chrome). Note that the title tag needs a closing tag, whereas meta doesn't

Finally we have the <body> tag. The body tag is a sibling to the head tag. The body tag is what contains all the content of the document

Structure

Structure is very important for HTML documents, otherwise there's no hierarchy in the doc. Structure is not just for visual aesthetics; it's also helpful for services to figure out what content is the most important. A helpful website for figuring your document online is https://gsnedders.html5.org/outliner/.

<p> One of the most basic, and common, tag is the paragraph tag. Each set of <p> tags creates a new paragraph containing the content within the tags. The line spacing between paragraphs can be adjusted using CSS. Technically, closing paragraph tags are optional in HTML5, but it's a bad practice to avoid

<h#> Another common structure tag is the header tag, where # is between 1 and 6. <h1> has the largest header font while <h6> is the smallest. A key mistake to avoid is to choose header types based on their size and font. Choose header #'s based on hierarchy, leave appearance to CSS

These are some useful tags for making webpages. Also probably handy for ENGL3844 assignments. These tags are examples of "semantic HTML", where the tags convey the actual meaning of what they are used for (tags like <div> and <span> are not semantic HTML). These tags are not automatically positioned by the browser, so you need to use CSS to put them where you want

Sectioning Elements

<h1> - <h6> Standard header tags

<article> Represents a part of the webpage that conveys particular information. Could be a combination of text, images, videos, and/or embeds. A good way to think about them is individaul elements of a page that can stand alone (don't need context to make sense)

<aside> Represents the information that is relevant to the page's content, typically in the form of a sidebar of a webpage. Content could be side notes, warnings, or information that will eventually be replaced in the future. aside elements should be nested within the section that the content is relevant to

<section> Defines a particular section of the HTML document. Multiple sections can exist in the same document. The difference between section and article is that section is used when you want content to be listed in a document outline, but it can't stand alone like an article could. Generally, you would use article for most sectioning purposes

<nav> Represents a section of a page that links to other pages or parts within a page. Navigation items like home, contact, and FAQs. Note that not all links have to be nav elements. In general, nav elements are only used for links within a site, not external sites

<div> A generic structuring tag that groups tags together. Should be used as a last resort when there's no other available tags that suffice for the grouping purpose. div has a class attribute where you can give a name to the group

<div class="book">
 <h5>The Hobbit</h5>
 <p>by J.R.R Tolkien</p>
 <p>A tiny human finds a ring and goes on an adventure</p>
</div

Semantic Elements

<main> Contains main sections of the HMTL document, like <header> and <footer> There should only be one main section in the entire document

<header> Defines the intro section of a web page (contains items such as logos, navigation, and themes). Usually goes hand-in-hand with an article section

<footer> Footers of a webpage, like quick links, copyright info, and other data that might be relevant to the entire website

Text Formatting

Most text formatting is done inline. Some formatting can also have affects on services like screen readers.

<i> Visual-only italics

<b> Visual-only bolding

<em>Emphasis italics (screen readers will also put verbal emphasis on the word)

<strong> Emphasis bolding (screen readers will also put verbal emphasis on the word)

<code> Simple in-line code formatting

<br> Does a line break (doesn't need a closing tag). Doesn't create a space between lines like <p> does

<pre> Preserves whitespace and indentation like it's written in the HTML document. Often used for code snippets since formatting has to be a certain way, so it'll typically display as monospace

<sub> Creates a subscript

<sup> Creates a superscript

<small> Makes test smaller, like fine print

Character References

HTML doesn't have escape characters, so if we want to display symbols (like angle brackets) without the browser interpreting it as actual code, we use character references. Some examples are:

&lt; is the < sign

&gt; is the > sign

&emsp; is a 4 space tab (like \t)

Lists

Lists are useful for displaying items, whether it's bullet points, numbered, or others

<li> General tag for a list item, regardless of list type

<ul> Unordered list (bullet points)

  • Item 1
  • Item 2
  • Item 3

<ol> Ordered list (numbered)

  1. Item 1
  2. Item 2
  3. Item 3

<dl> Definition (or description) list, where each item has a term and definition (think key-value pairs)

<dt>Term name</dt>
<dd>Term definition</dd>
<dd>Additional definitions if needed<dd>
<dd>...<dd>

Example list code:

<ul>
  <li>list item 1</li>
  <li>list item 2</li>
  <li>list item 3</li>
</ul>

Lists can be nested to create sublists by creating a new list within a list item <li>, like so:

<li>
  <ul>
    <li>list item 1</li>
    <li>list item 2</li>
    <li>list item 3</li>
  </ul>
</li>

Ordered lists also have some fun atrributes:

start indicates which number or letter that the list should start with. For example, doing <ol start="6"> will have the list start with 6,7,8,...

reversed will reverse the ordering, with the bottom as the starting point (<ol reversed>)

type changes the order notation (numbers, roman numerals, letters, etc.) like so: <ol type="a">

Media

Images


When a browser encounters an image tag, it replaces that tag with an image based on the information given to it. Therefore it's really important to maintain the correct attributes. The image tag is actually an inline tag, so it needs to be contained within paragraphs or other structure tags. Here are some attributes that are important:

  • src Indicates the source of the image. Usually the relative path to the image file in the directory that the HTML file is in. The only non-optional attribute

  • width Width of image in pixels. If not specified, the browser will display the image at its default size. However, it could cause the image to be disorted, so best to specify

  • height Height in pixels. Same practice as width

  • alt Gives a short description to the image

  • srcset Creates a responsive image based on screen resolution or width

Note that the img tag doesn't need a closing tag. To change image alignment, it's best to use CSS. There is a deprecated align attribute, and while most browsers still support it, CSS is the best way to go.

To add captions to figure we need to use 2 elements: figcaption and figure. The figcaption element is a sibling with img, with the figure wrapping the two.

Audio

Similar to images, we have an <audio> element with a src attribute that points to the location of the file (either local file or external source). There are other useful attributes:

  • loop audio automatically loops back to beginning

  • autoplay audio automatically plays on page load

WAI-ARIA Roles

ARIA roles are used for improved accessibility with the role attribute in tags. This helps things like screen readers to better understand the structure of our article. Some examples include:

  • banner site-oriented content like logos

  • main main content of the page

  • navigation page navigation tools

  • complementary extra information for the page like aside

Note that some roles have the exact same name as their tags which seems like overkill. However, since they're fairly new and not all browsers/tools support roles, it's best to use both for now.

Links

Links are super important for HTML pages and the Internet. They can be linked to external pages, other HTML pages, and even sections within the page. Links use the anchor element: <a>, which is an inline element. The general form is <a href="link" title="title">. href is the link to the destination. title provides a short description of where the link leads to, useful for assistive tech like screen readers.

<a href="path to HTML file"> is used to redirect to other HTML documents. href is the relative path to the document (where we use ../ and whatnot). Something we can do is just specify the folder of the relative path. The browser will then automatically look for an index.html file to display.

<a href="link to external page"> can be used to redirect to external websites. Make sure it's the full link (starts with http or https), not just "www"

<a href="path to download file" download> is used for downloadable content. Normally, if a browser can't display a particular document (where href points to some file in the directory like .zip or .exe), it will automatically download it. However, if the browser can display it (.html or .pdf), then it'll actually display it. In that case, we need to explicitly specify for it to be downloaded with the download attribute. If we want to rename the file for downloading, we can set the atrribute equal to the new name: download="new name"

<a href="path to HTML file"> lets us jump to sections within the document. Before setting the anchor element, we need to add the id attribute to headers of the sections. This ID is called a fragment identifier lets the anchor element identify which section to jump to: <h1 id="one">. Afterwards, we set the href attribute to #[section id] like so: <a href="#one">. To link to regions in other pages, you just need to find the fragment identifier of that page and use it like so: <a href="[filename]#[fragment id]">. For example: <a href="intro.html#me"> will jump to the "me" section in "intro.html". Same thing applies to external websites, just append the fragment id to the full link with the # (this is why the url's for some specific webpages contain # signs, like Wikipedia)

CSS

CSS is great for formatting blah blah blah...The general form is "[property]:[value];

One way to use CSS is to use the style attribute. You can then set properties like color and font-weight so: <p style="color:red;font-weight:normal;">. However, this can be a pain to manually set and update them

A more efficient way is to use the style element in the head section. Afterwards, you can change the appearance for each element (to specify a global default, set the properties under body):

<style>
 h2 {color:red; font-weight:normal;}
<style>
(this can also be expanded into a code block style)

Can also use it for class indicators. Instead of h2, we can do .[indicator] { properties } where every element with the class="indicator" attribute. We can also do [element].[indicator] { properties } where both element and class indicator has to match

Some of the properties we can set are:

  • font-family font like Georgia, Times New Roman, etc.
  • font-size where you can specifiy it with [x]em where em is the default font and x is the factor. This is more dynamic than setting pixels, since it adjusts the relative size (16p is small on computer, massive on phone). font-size: 2em; means twice the default font size
  • font-style bold, italic
  • line-height spacing between
  • margin
  • ...and more!

To set color, you use hex codes. Some atrributes include:

  • background-color (background color of entire document)
  • background (background of specific elements)
  • border-bottom

So far, all these changes only apply to the document they're in. In order to externalize these styles (apply styles to every HTML doc in workspace/webpage, we need to create a .css file. Basically, move all the code under the style elements into the CSS file. Replace the style element with a link element:

<link rel="stylesheet" href="link to css file">

JavaScript

JavaScript adds interactivity to HTML documents, like buttons and drop-down menus. To add a JS script, use the <script> element. This is generally a standalone element (not nested anywhere) contained within the body section.

One script we can set up is window.onload. This function will run once the browser window is done loading. We would do something like:

window.onload = function() {
 // insert code
}

To get elements from the document, we can use the document object and its functions like getElementById() or getElementsByTagName()

To externalize the JS scripts, export them into a .js file. Then we change the <script> element by adding a src="path to js file" attribute. Remember to add a closing script tag!
